On any content page you can create HTML forms that can either pass data back to the database, generate an e-mail, or both. To use this functionality, just write the following HTML on any content page where you would like to create a form.

Form Syntax:
    <form method="POST" action="htmlformresults.aspx">
    </form>

Between the <form></form> tags there are several parameters that you can provide to modify how the HTML forms will post information to the server. The full list of input parameters and their uses is shown below.

Input Commands:

EmailTo E-Mail address to send form results to. If not set, no e-mail will be generated.
EmailFrom E-Mail address in the From field. If not set, your default e-mail will be used.
EmailSubject The subject line of the generated e-mail.
EmailMessage The message sent to the e-mail recipient.
EmailShowResults If set to 1, the full results of the form will be sent with the e-mail. If set to 0, only the EmailMessage will be sent.
FormName Form name used when posting results to the server.
FormText Message to be placed above the form results when posted back to the server.
SubmitMessage The message your users will see after submitting a form.
PostToDatabase If set to 1, the data will be posted to your database, which can be viewed on the admin screen. If set to 0, only an e-mail will be generated if EmailTo is set, otherwise the form will not submit any data.
Submit The ID or NAME value of the submit button must be "Submit"
ANY_FIELD_NAME Any other field ID or NAME used besides the values mentioned above will be used as part of the data set. You can add as many fields as you would like.
Required-XXXXX Replace the XXXXX with the name of any one of your fields and set the value to 1 to make the field a required field. For example, "Required-Username" = 1 would make the "Username" field required.

Example Form:
    <form method="POST" action="htmlformresults.aspx">

    <input type="hidden" name="EmailTo" value="you@yourdomain.com" />
    <input type="hidden" name="EmailSubject" value="A User Has Signed Up!" />
    <input type="hidden" name="EmailMessage" value="Visit the admin screen to view the data." />
    <input type="hidden" name="EmailShowResults" value="0" />
    <input type="hidden" name="Required-User Name" value="1" />

    Your Name: <input name="User Name" type="text" size="30" /><br />
    Your E-Mail: <input name="User E-Mail" type="text" size="30" /><br />
    <input name="submit" type="submit" value="Submit" />

    </form>

The form above will generate an e-mail to "you@yourdomain.com" with the subject "A User Has Signed Up!" The e-mail will contain the text "Visit the admin screen to view the data." The "User Name" and "User E-Mail" values will not be sent in the e-mail. They will be posted to the HTML Forms section of the admin console. The "User Name" value is a required field.
